# Structorex

[![Python Version](https://img.shields.io/badge/python-3.8%2B-blue.svg)](https://www.python.org/downloads/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

**Structorex** — это мощный инструмент командной строки, который генерирует подробный отчет о структуре файлов вашего проекта и их содержимом. Он обеспечивает визуальное представление иерархии каталогов и позволяет просматривать содержимое файлов прямо в сгенерированном отчете. Идеально подходит для создания контекста для LLM (ИИ), передачи проектов и документирования.

## ✨ Особенности

- 🌳 **Визуальная структура дерева:** Наглядное представление структуры каталогов.
- 📄 **Просмотр содержимого:** Встраивает исходный код и текст прямо в отчет.
- ⚙️ **Мощный CLI:** Полностью настраивается через аргументы командной строки (`argparse`).
- 🔒 **Безопасность:** Автоматически игнорирует конфиденциальные файлы, такие как `.env`, `id_rsa`, `.pem`, `secrets.*`.
- 🚦 **Поддержка `.gitignore`:** Уважает правила вашего `.gitignore` из коробки (использует `pathspec`).
- 📝 **Множество форматов:** Экспорт в форматах Text, Markdown, JSON и интерактивном HTML.
- 📦 **Экспорт ZIP-снимка:** Создает чистый ZIP-архив файлов вашего проекта (без игнорируемых).
- 🔣 **Умная кодировка:** Автоматически определяет кодировку текста (UTF-8, ISO-8859-1 и др.) с помощью `charset-normalizer`.
- ⚡ **Высокая производительность:** Параллельное чтение файлов (`ThreadPoolExecutor`) с индикатором выполнения (`tqdm`).
- 🛡️ **Фильтрация бинарных файлов:** Автоматически пропускает изображения, видео, аудио и скомпилированные файлы.

## 🚀 Установка

Установите через PyPI (рекомендуется):
```bash
pip install structorex
```

Или из исходного кода:
```bash
git clone https://github.com/yourusername/structorex.git
cd structorex
pip install -e .
```

## 💻 Использование

### Основные команды

Проанализировать текущий каталог и создать текстовый отчет:
```bash
structorex
```

Создать отчет в формате **Markdown** (отлично подходит для GitHub и ИИ):
```bash
structorex . --format markdown --output project_report.md
```

Создать **интерактивный HTML**-отчет (с кликабельным деревом):
```bash
structorex . --format html --output index.html
```

Сделать чистый **ZIP-снимок** вашего кода:
```bash
structorex . --export-zip clean_source_code.zip
```

### Продвинутая конфигурация

Structorex предоставляет гибкие аргументы для точной настройки:
```bash
structorex /path/to/project -o output.json -f json --exclude node_modules dist --max-size 5
```

**Доступные параметры:**
- `-h`, `--help`: Показать справку.
- `-o`, `--output`: Имя выходного файла (по умолчанию: `project_report.txt`).
- `-f`, `--format`: Формат вывода (`text`, `markdown`, `json`, `html`).
- `-e`, `--exclude`: Дополнительные папки для исключения.
- `-a`, `--allowed`: Переопределить разрешенные расширения файлов (например, `.py` `.txt`).
- `-m`, `--max-size`: Максимальный размер файла в МБ (по умолчанию 10 МБ).
- `-v`, `--verbose`: Включить подробное логирование.
- `-s`, `--show-skipped`: Показывать пропущенные файлы в дереве как `[skipped]`.
- `-z`, `--export-zip`: Создать ZIP-архив отфильтрованного проекта.

## ⚙️ Файлы конфигурации

Вместо того чтобы каждый раз вводить аргументы, Structorex автоматически загрузит настройки, если в корне проекта есть один из этих файлов:

**`.structorex.yaml`**
```yaml
format: markdown
output: summary.md
exclude:
  - tests
  - docs
max_size: 2
```

**`pyproject.toml`**
```toml
[tool.structorex]
format = "json"
output = "analysis.json"
exclude = ["build", ".tox"]
```

```text
├── pyproject.toml
└── .gitignore
```
