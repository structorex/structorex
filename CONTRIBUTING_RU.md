# Участие в разработке Structorex

Прежде всего, спасибо, что рассматриваете возможность внести свой вклад в Structorex! Именно благодаря таким людям Structorex становится отличным инструментом.

## Как вы можете помочь

### 1. Сообщить об ошибке или предложить функцию
Если вы нашли ошибку или у вас есть идея для новой функции, пожалуйста, создайте Issue на GitHub. Укажите как можно больше подробностей:
- Шаги для воспроизведения ошибки
- Ожидаемое и фактическое поведение
- Ваша операционная система и версия Python

### 2. Отправить Pull Request
Если вы хотите исправить ошибку или добавить функцию самостоятельно:
1. Сделайте форк репозитория и клонируйте его на свой компьютер.
2. Создайте новую ветку для вашей функции (`git checkout -b feature/amazing-feature`).
3. Внесите изменения и добавьте тесты, если это применимо.
4. Запустите тесты и линтеры (`pytest`, `ruff`, `mypy`).
5. Сделайте коммит с понятным и описательным сообщением.
6. Отправьте ветку (push) и создайте Pull Request.

## Настройка среды разработки

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/structorex.git
   cd structorex
   ```
2. Установите в режиме разработки со всеми зависимостями:
   ```bash
   pip install -e .[dev]
   ```
3. Запустите тесты, чтобы убедиться, что все работает:
   ```bash
   pytest
   ```

## Правила написания кода
- Следуйте стандарту **PEP 8** для кода на Python.
- Пишите docstrings для всех публичных классов, методов и функций.
- Добавляйте аннотации типов (Type Hints) для лучшей читаемости.
- Помните о производительности (используйте генераторы, ленивые вычисления, многопоточность, где это уместно).

Спасибо за вашу поддержку!
