# Structorex - Официальное руководство пользователя 🚀

Добро пожаловать в **Structorex**! Этот инструмент поможет вам за секунды прочитать целые проекты, визуализировать их файловую структуру и безопасно экспортировать содержимое файлов.

Идеально подходит для передачи контекста моделям ИИ (таким как ChatGPT или Claude), создания документации по проекту или чистых резервных копий исходного кода.

---

## 🛠️ Установка

Убедитесь, что у вас установлен Python версии 3.8 или выше.
Установите инструмент глобально в вашей системе:

```bash
pip install structorex
```

---

## 📖 1. Быстрый старт

Откройте терминал, перейдите в папку вашего проекта и выполните команду:

```bash
structorex
```

**Что произойдет?**
1. Structorex просканирует все папки и файлы.
2. Он автоматически проигнорирует неважные файлы (изображения, `.git`, `node_modules`).
3. Он безопасно отфильтрует конфиденциальные данные (`.env`, `id_rsa`, `.pem`).
4. Он создаст файл `project_report.txt`, содержащий чистое дерево файлов сверху и весь исходный код снизу.

Если вы хотите просканировать другой каталог без перехода в него:
```bash
structorex /path/to/my/project
```

---

## 🤖 2. Передача кода в ИИ / LLM (Markdown)

ИИ лучше всего понимают формат **Markdown** (`.md`). В Structorex есть специальный режим, который форматирует ваш код с правильной подсветкой синтаксиса.

```bash
structorex . --format markdown --output my_ai_prompt.md
```
Откройте файл `my_ai_prompt.md`, скопируйте всё содержимое (Ctrl+A, Ctrl+C) и вставьте прямо в чат с ИИ. ИИ мгновенно поймет всю архитектуру вашего проекта!

---

## 🌐 3. Интерактивные HTML-отчеты

Предпочитаете изучать свой проект прямо в браузере? Сгенерируйте интерактивную веб-страницу:

```bash
structorex . --format html --output project.html
```
Дважды щелкните по `project.html`, чтобы открыть его. Слева вы увидите кликабельное оглавление. Нажмите на любой файл, и его исходный код появится справа!

---

## 📦 4. Экспорт чистого ZIP-снимка

Часто нужно поделиться исходным кодом, но без лишнего мусора (`node_modules`, скомпилированные файлы, пароли). Structorex может создать для вас чистую, отфильтрованную резервную копию:

```bash
structorex . --export-zip clean_code.zip
```
В результате получится небольшой чистый ZIP-файл, содержащий только ваш исходный код.

---

## 🛡️ 5. Что именно фильтруется?

Structorex умный. По умолчанию он пропускает файлы, которые вы не хотите видеть в отчетах:
1. **`.gitignore`**: Все, что указано в вашем `.gitignore`, игнорируется.
2. **Секреты**: Такие файлы, как `.env`, `secrets.json`, `.pem`, **всегда блокируются** в целях безопасности.
3. **Большие файлы**: Файлы размером более 10 МБ пропускаются (можно изменить с помощью `--max-size 50`).
4. **Бинарные файлы**: Изображения, видео, `.exe` или PDF игнорируются автоматически.

Если вы хотите **дополнительно** исключить определенные папки:
```bash
structorex . --exclude frontend/build docs tests
```

---

## ⚙️ 6. Постоянная конфигурация

Если вы постоянно используете одни и те же настройки в проекте (например, всегда экспортируете в HTML, всегда исключаете папку X), вам не нужно каждый раз вводить аргументы.

Создайте файл `.structorex.yaml` в папке вашего проекта:

```yaml
format: html
output: documentation.html
exclude:
  - tests
  - old_backend
max_size: 5
```
Теперь, просто введя `structorex`, инструмент автоматически загрузит эти настройки!

*(Примечание: Python-разработчики могут добавить это в свой `pyproject.toml` в раздел `[tool.structorex]`).*
